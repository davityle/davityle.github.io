Why Im Not Switching to Firefox

I’ve been very excited with the progress that Firefox has been making in recent times and have been contemplating making the switch from Chrome for a while. The release of Firefox 57 nudged me over the edge and I excitedly downloaded Firefox Developer Edition like I an imaginative child on Christmas morning. I took some time to customize it to my liking and then proceeded to use it as my primary browser over the weekend. It was awesome, and I was super stoked.

Why Firefox 57 is amazing

  • It is surprisingly fast - I was a bit skeptical about the claims about how fast Firefox 57 is. I was wrong to be. It is noticeably faster.
  • It is customizable - coming from Chrome I didn’t realize the flexibility that Firefox offers. After just a few hours I had a setup that I was very happy with. I used custom themes and added the Tree Style Tab plugin.
  • It is privacy senstitive.
  • It is open source.
  • It isn’t Chrome - as a previously self proclaimed Google fan boy this is a wierd one to see myself typing out but, I don’t think I’m the only one that is excited to see some real competetion against Google in the browser space.

All of this begs the question.

Why I’m not switching to Firefox

  • The dev tools are unusable for my workflow

That is just about it. I work on Lucidchart’s editor — which isn’t the lightest application on the web. Especially pre optimization, and minification — the environment in which we develop. Using the devtools in Chrome while working on the editor works mostly as expected. There have been a few bugs around source maps but I have just turned those off and everything works great. In Firefox 57 Developer Edition, they are completely unusable.

Perhaps even worse. With the dev tools open, I can no longer load the Lucidchart editor while developing. Opening the dev tools and then loading the editor completely locked up all of Firefox except for, ironically, the dev tools. I only ever waited about 5 minutes before I gave up completely.

Surprisingly, there is no problem loading the minified/optimized editor in production. You can try it out for yourself using the demo editor.

I spend enough of my day working on the editor with the dev tools open that this is a deal breaker for me. Hopefully, Mozilla can adress these problems soon.

Conclusion

The latest versions of Firefox are amazing and I’m stoked to see what is coming next. Unfortunately, the dev tools are unusable for what I need to use them for so I can’t make the switch …. yet.
